NAME

pstree − display a tree of processes

SYNOPSIS

pstree [-A | --all] [-a | --arguments] [-C | --cpu] [--color] [-d | --debug] [-g | --show-pgids] [-G | --age] [-h | --help] [-T | --hide-threads] [-i | --ibm-850] [-I | --uid-transitions] [-k | --color-attr attr] [-l | --level level] [-m | --memory] [-c | --compact-not] [-o | --order-by field] [--show-group] [-O | --show-owner] [-p | --show-pids] [-P | --pid PID] [-q | --color-scheme scheme] [-r | --rainbow] [-s | --contains pattern] [-S | --show-pgls] [--show-ppids] [-t | --threads] [-u | --utf-8] [-U | --user-transitions] [--user username] [-v | --vt-100] [-V | --version] [-w | --wide] [-X | --exclude-root]

DESCRIPTION

pstree shows running processes as a tree. The tree is rooted at launchd. If --pid is specified, the tree will be rooted at the specified PID. If --user is specified, all process trees rooted at processes owned by that user are shown. By default, processes are grouped and collapsed by command name and arguments. This can be disabled with the --compact-not option.

OPTIONS

−G, −-age

Show the age of each process in the list using the format (dd:hh:mm:ss).

−A, −-all

Equivalent to --show-owner --show-group --show-pids --show-pgids --age --cpu --memory --threads --arguments.

−a, −-arguments

Show command line arguments after the process name.

−-color

Colorize the pstree output. This option is only visible in the help if your terminal supports color. This option is not available if your terminal doesn’t support at least 8 color output. This option cannot be used with --color-attr or --rainbow.

−k, −-color-attr attr

Color the process entry by the given attribute. Valid options are: age, cpu, mem. This option is not available if your terminal doesn’t support at least 8 color output. This option implies --compact-not, since some of the results may be buried in collapsed subtrees. This option cannot be used with --color or --rainbow.

age

Colors processes by their age: red (<1 minute), yellow (1 minute to 1 hour), cyan (1 hour to 1 day), green (>1 day).

cpu

Colors processes by CPU usage: green (<5%), yellow (5-15%), red (>15%).

mem

Colors processes by memory usage as percentage of total system memory: green (<10%), yellow (10-20%), red (>20%).

−q, −-color-scheme scheme

Override the default color scheme. Valid options are: darwin, linux, powershell, windows10, xterm.

−c, −-compact-not

Do not compact identical subtrees in output. By default, identical process subtrees are shown only once with a count indicating how many instances exist (e.g., "process---N*[process]"). This option disables compaction, showing each process individually.

−s, −-contains pattern

Show only branches containing processes with pattern in the command line. This option implies --compact-not, since some of the results may be buried in collapsed subtrees.

−C, −-cpu

Show the CPU utilization for each process in the list using the format (c:0.00%). This option implies --compact-not, since some of the results may be buried in collapsed subtrees.

−d, −-debug

Show debugging data. Specify multiple times (-d, -dd, -ddd) to increase debugging level.

−X, −-exclude-root

Don’t show branches containing only root processes. This option cannot be used with --user.

−h, −-help

Display a help message and exit.

−T, −-hide-threads

Hide threads, showing only processes. When this option is used, threads will not be displayed in the process tree and thread information will be omitted from compact mode output. Thread counts will still be shown if the -t option is used. This option is only supported on Linux OSes. This is because Darwin OS does not expose thread IDs.

−i, −-ibm-850

Use IBM-850 line drawing characters; only supported on DOS/Windows.

−l, −-level level

Print tree to level level deep.

−m, −-memory

Show the memory usage for each process in the list using the format (m:0.00 MiB). This option implies --compact-not, since some of the results may be buried in collapsed subtrees.

−o, −-order-by field

Sort processes in ascending order by a given field. Available options are: age, cmd, cpu, mem, pid, threads, user.

−P, −-pid PID

Show only branches containing process PID.

−r, −-rainbow

Display the output in a really annoying rainbow pattern. This option is not available if your terminal doesn’t support 256 color output. This option cannot be used with --color or --color-attr.

−g, −-show-pgids

Show PGIDs. Process Group IDs are shown as decimal numbers in parentheses after each process name.

−S, −-show-pgls

Show process group leader indicators. By default, process group leaders are not marked with special characters in the output.

−p, −-show-pids

Show PIDs. Process IDs are shown as decimal numbers in parentheses after each process name. If the display of threads is enabled (Linux-only), this option will display the thread ID as a decimal number in parentheses after each thread name.

−-show-ppids

Show parent process IDs. Parent Process IDs are shown as decimal numbers in parentheses after each process name.

−-show-group

Show the group of the process.

−O, −-show-owner

Show the owner of the process.

−t, −-threads

Show the number of threads for each process in the list using the format (t:xx).

−I, −-uid-transitions

Show processes where the user ID changes from the parent process, e.g., (uid→uid). This makes it easy to identify privilege escalations or drops in the process tree. This option cannot be used with --user-transitions.

−-user user

Show only branches containing processes of user. This option can be used more than once. This option cannot be used with --exclude-root.

−U, −-user-transitions

Show processes where the username changes from the parent process, e.g., (user→user). This makes it easy to identify privilege escalations or drops in the process tree. This option cannot be used with --uid-transitions.

−u, −-utf-8

Use UTF-8 (Unicode) line drawing characters.

−V, −-version

Display version information and exit.

−v, −-vt-100

Use VT-100 line drawing characters.

−w, −-wide

Do not truncate output to the width of the screen.

EXAMPLES

Display a basic process tree:

pstree

Show a process tree with PIDs, CPU usage, memory usage, and process owners:

pstree -p -c -m -O

Show a process tree for a specific user with process age and thread counts:

pstree --user username -G -t

Show a process tree rooted at a specific process with all details:

pstree -P 1234 -A

Show a process tree with UTF-8 graphics and process group leader indicators:

pstree -u -S

Show only processes containing a specific pattern in the command line:

pstree -s "firefox"
